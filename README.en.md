# Next.js API Template

This is a template for creating RESTful APIs with Next.js, MongoDB, and Swagger. It provides a complete CRUD (Create, Read, Update, Delete) functionality for a generic "Item" model, and a ready-to-use interactive documentation page.

## Features

- **Next.js API Routes**: Backend API built entirely within the Next.js framework.
- **MongoDB Integration**: Uses Mongoose for elegant MongoDB object modeling.
- **Swagger/OpenAPI Documentation**: Interactive API documentation powered by Swagger UI.
- **TypeScript**: Fully typed codebase.
- **Environment-based Configuration**: Securely manage your database connection strings.

## Getting Started

### Prerequisites

- Node.js (v16 or later)
- npm or yarn
- A MongoDB database (you can get a free one from [MongoDB Atlas](https://www.mongodb.com/cloud/atlas))

### Installation

1. **Clone the repository:**

   ```bash
   git clone https://github.com/your-username/nextjs-api-template.git
   cd nextjs-api-template
   ```

2. **Install dependencies:**

   ```bash
   npm install
   ```

3. **Set up environment variables:**

   Create a file named `.env.local` in the root of the project and add your MongoDB connection string:

   ```
   MONGODB_URI=your_mongodb_connection_string
   ```

4. **Run the development server:**

   ```bash
   npm run dev
   ```

   The application will be available at `http://localhost:3000`.

## API Documentation

To view the interactive API documentation, navigate to:

[http://localhost:3000/docs](http://localhost:3000/docs)

This page is generated by Swagger UI and provides a detailed overview of all available endpoints, their parameters, and expected responses. You can even test the API directly from this page.

## Available API Endpoints

- `GET /api/items`: Get all items.
- `POST /api/items`: Create a new item.
- `GET /api/items/{id}`: Get a single item by its ID.
- `PUT /api/items/{id}`: Update an item by its ID.
- `DELETE /api/items/{id}`: Delete an item by its ID.

## Customization

To adapt this template for your own needs, you'll primarily need to:

1.  **Modify the Mongoose Model**:
    -   Edit `src/models/Item.ts` to define your own data schema.

2.  **Update the API Endpoints**:
    -   Change the logic in `src/pages/api/items/` to match your new model.

3.  **Update Swagger Documentation**:
    -   Modify the JSDoc comments in the API route files to reflect your changes.
    -   Update the schema definition in `swagger.js`.
